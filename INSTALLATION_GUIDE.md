# 🔧 Installation & Deployment Guide

## Complete Step-by-Step Instructions

---

## 📋 Before You Start

**You Will Need:**
- A modern web browser (Chrome, Firefox, Safari, or Edge)
- Internet connection (for deployment only)
- Optional: GitHub account (if using GitHub Pages)

**Time Required:**
- Local use: 1 minute
- GitHub Pages deployment: 15 minutes
- Other hosting: 5-10 minutes

---

## Option 1: Use Locally (Fastest)

### Steps:

1. **Download** the file `AQAR_OER_Complete.html`
   - Right-click → Save as
   - Or download from GitHub release

2. **Save** to your computer
   - Desktop, Downloads folder, anywhere

3. **Open** the file
   - Double-click the HTML file
   - Or right-click → Open with → Browser

4. **Use immediately!**
   - Select role
   - Enter name
   - Start filling forms

### Advantages:
- ✅ Instant setup
- ✅ Works offline after loading
- ✅ No internet needed (after first load)
- ✅ Maximum privacy

### Limitations:
- Single user at a time
- Data not synced across computers
- Share via email/USB

## Option 2: Deploy on GitHub Pages (Recommended)

### Advantages:
- ✅ Free forever
- ✅ Live link everyone can access
- ✅ Professional hosting
- ✅ Easy to update

### Step-by-Step:

#### Step 1: Create GitHub Account (5 minutes)

1. Go to **https://github.com**
2. Click **"Sign up"**
3. Enter email address
4. Create strong password
5. Choose username (e.g., `aqar-system-skr`)
6. Verify email
7. ✓ Account created!

#### Step 2: Create Repository (3 minutes)

1. After login, click **"+"** (top right)
2. Select **"New repository"**
3. Fill in details:
   - **Repository name**: `aqar-system`
   - **Description**: "AQAR Management System - OER"
   - **Public**: ✓ Check this box
   - **Initialize with README**: Uncheck

4. Click **"Create repository"**

#### Step 3: Upload Files (5 minutes)

1. In repository, click **"Add file"** → **"Upload files"**
2. Drag & drop OR click to select:
   - `AQAR_OER_Complete.html` ⭐ (Main file)
   - `README_GITHUB.md`
   - `LICENSE_CC_BY_40.md`
   - Other documentation files

3. Click **"Commit changes"**
4. Wait for upload to finish

#### Step 4: Enable GitHub Pages (3 minutes)

1. Click **"Settings"** (top navigation)
2. Click **"Pages"** (left sidebar)
3. Under "Source":
   - **Branch**: Select `main`
   - **Folder**: Select `/ (root)`

4. Click **"Save"**
5. Wait 1-2 minutes

#### Step 5: Get Your Live Link

After Pages is enabled, you'll see:

```
Your site is live at:
https://YOUR_USERNAME.github.io/aqar-system/
```

**Full URL to share:**
```
https://YOUR_USERNAME.github.io/aqar-system/AQAR_OER_Complete.html
```

## Troubleshooting

### Problem: File won't open

**Solution:**
- Try different browser (Chrome recommended)
- Download file again
- Check file name: `AQAR_OER_Complete.html`
- Make sure you have internet connection

### Problem: GitHub Pages shows 404

**Solution:**
1. Verify file is uploaded (check repository)
2. Check Pages is enabled (Settings → Pages)
3. Wait 5 minutes and refresh
4. Try incognito/private browsing
5. Clear browser cache

### Problem: Data not saving

**Solution:**
1. Check browser allows localStorage (enable in settings)
2. Try different browser
3. Clear browser cache
4. Check you have free disk space
5. Download and backup data regularly

### Problem: Can't upload to GitHub

**Solution:**
1. Check GitHub account is logged in
2. Check repository is public
3. Try using GitHub Desktop app
4. Check file size (HTML is usually small)
5. Try uploading one file at a time

### Problem: Slow loading on mobile

**Solution:**
1. Check internet connection is good
2. Try different browser
3. Close other apps using data
4. Refresh page (Ctrl+R or Cmd+R)
5. Wait a few seconds after loading

## After Deployment

### Important Setup:

1. **Test the system**
   - Try logging in with different roles
   - Submit sample data
   - Check dashboard updates

2. **Backup data regularly**
   - Every Friday, download data
   - Store in cloud (Google Drive, Dropbox)
   - Keep multiple copies

3. **Update if needed**
   - Edit file in GitHub
   - Changes go live automatically
   - Test before announcing

4. **Monitor submissions**
   - Log in as IQAC regularly
   - Track who has submitted
   - Send reminders for pending

## Advanced: Custom Domain

Want your own domain (e.g., `aqar.yourschool.edu`)?

1. **Buy domain**: GoDaddy, Namecheap, Google Domains
2. **In GitHub Settings → Pages**:
   - Enter domain name
   - Add DNS records (GitHub will show instructions)
3. **Wait 24-48 hours** for DNS to update
